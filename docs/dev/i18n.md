# i18n 翻译文件结构

本目录包含了 Cocos CLI 的国际化翻译文件，按照模块化的方式组织，目前初始化放置在 src 最外层，供 src 内各个模块调用。

## 目录结构

```text
static/i18n/
├── zh/                    # 中文翻译
│   ├── assets.json       # 资源数据库相关翻译
│   ├── builder.json      # 构建器相关翻译（包含平台特定翻译）
│   ├── common.json       # 通用翻译
│   └── importer.json     # 导入器相关翻译
└── en/                    # 英文翻译
    ├── assets.json
    ├── builder.json
    ├── common.json
    └── importer.json
```

## 文件命名规则

- 文件名基于 i18n key 的第一个部分（如 `asset-db.saveAsset.fail` → `asset-db.json`）
- 每个文件包含该模块下的所有翻译 key
- 使用嵌套的 JSON 对象结构来组织翻译内容

## 翻译 Key 映射

### assets.json

- `assets.*` - 资源数据库相关功能
- `assets.saveAssetMeta.*` - 资源 META 保存相关功能

### builder.json  

- `builder.*` - 构建器相关功能
- `builder.platforms.native.*` - 原生平台相关功能
- `builder.platforms.mac.*` - Mac 平台特定功能

### importer.json

- `importer.*` - 导入器相关功能
- `importer.gltf.*` - glTF 导入器相关功能
- `importer.fbx.*` - FBX 导入器相关功能
- `importer.dragonbones_atlas.*` - DragonBones 图集相关功能
- `importer.script.*` - 脚本导入器相关功能

### common.json

- 通用 UI 文本和常用词汇

## 使用方式

在代码中使用：

```typescript
import i18n from '../i18n';

// 使用翻译
const message = i18n.t('assets.saveAsset.fail.asset');
const title = i18n.t('common.save');
```

## 添加新翻译

1. 确定 key 的第一个部分（如 `new-module.someKey` → `new-module.json`）
2. 在对应的语言目录下创建或更新 JSON 文件
3. 使用嵌套结构组织翻译内容
4. 同时更新所有支持的语言版本
